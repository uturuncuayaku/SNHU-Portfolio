# Final Project Scenario
Working as an AI developer for a company, I've been tasked with developing a treasure hunting game using Deep Q-Learning. This pathfinding problem using a pirate and a gold stash along a maze is solved by training the pirate on finding the optimal path towards the gold and then competing against another player.

What do computer scientists do and why does it matter?  

Computer science is about problem solving through abstraction. It often involves
hardware and software tools that help humans design algorithms, architectures,
and software systems that translate human needs into machine execution.
It matters because computation has become "universal solvent" for modern
challenges. This means computer science expand the boundaries of what is
possible for society to achieve. 

How do you approach problems as a computer scientist?  

I approach problems as a computer scientist by deconstructing the black box of
technology. I think about the hardware, firmware, and kernel layer understanding
that every failure as a point in a system that can be isolated, tested and
understood through research and effective communication. I also like to think
about state machines and handshakes because a process usually has an
initialization, a starting point, a clean up or joining function and an ending
point that directly translates the real world into machine level concepts.
Handshakes are also imperative as they establish communication avenues across
networks or protocol stacks within a system and this is vital for many data
intensive applications to share or communicate functionalities across different
mediums or tasks.

What are my ethical responsibilities to the end user and the organization?  

Ethical responsibilities that I find to be important to the end user and an
organization are to prioritize safety and reliability. For instance, robots
could cause physical harm or machine learning algorithms could have bad data
that causes incorrect feedback for sensitive applications within banking or
employment. There is also a need for data privacy and soverignty which means
user data is secure and anonymized. Also, thinking about different types of
users that may need more inclusive devices or software through targeted
accessibility concerns that promote ease of use for people with different
physical or cognitive needs. This adoption of transparent and  honest design are
critical to the success of organizational needs that value secure by design code
choices and efficient stewardship of resources.
